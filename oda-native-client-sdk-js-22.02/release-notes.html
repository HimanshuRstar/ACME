Oracle Digital Assistant Native Client SDK for Web

Release 22.02

We are thrilled to share the 22.02 release of the Oracle Digital Assistant Native Client SDK for Web. This time it's refreshed with a new modern design with UX improvements and support for default client responses.

What's New

  • Redwood Design - We're happy to introduce the new Oracle standard for conversation design. We've redesigned the chat experience from scratch to improve the usability and the look and feel of the interface while still retaining the customization that we've always provided.

    Here are some highlights:

    • Header Actions - To keep the chat header clean and focused, the chat header actions are now collapsed into a menu button when there are three or more action buttons.

      The multi lang selection option has been converted from a select input to a dropdown menu to free up space. When in the collapsed mode, the language selection appears as a sub-menu of the header actions menu.

      You can disable this feature by setting enableHeaderActionCollapse to false.

    • Message Blocks - To eliminate repetitive information and keep the transcript area clean, all message bubbles from the same sender (the skill or the user) that are sent within 10 seconds of each other are organized into a message block with a single avatar and timestamp.
    • Timestamps - We are expanding the timestampMode to support more possible values of none, default, relative (the default mode) and absolute.

      If you don't want to display any timestamps, you can set the timestampMode to none. Because this eliminates the need of setting the additional enableTimestamp setting, we have deprecated the enableTimestamp setting.

      We have also updated the default timestamp format to a shorter form. For example: Thu Mar 10, 11:27 AM. We have also removed the read check mark in the absolute timestamp mode because there's no need for this additional marker when users can already see that messages have been read.

  • Default Client Responses - We now support default client-side responses accompanied by the typing indicator when either the skill response has been delayed, or when there is no response from the skill at all.

    This feature is controlled by the enableDefaultClientResponse boolean flag, which is set to false by default.

    If the user sends out the first message/query, but the skill (ODA) is not able to respond within the number of seconds set by the defaultGreetingTimeout flag, the client displays a default greeting message that can be configured using the defaultGreetingMessage translation string.

    Next, the client checks again for the skill response. The client displays the skill's response if it has been received, but if it hasn't, then the client displays a wait message (configured with the defaultWaitMessageInterval translation string) at intervals set by the defaultWaitMessageInterval flag.

    When the wait for the skill response exceeds the threshold set by the typingIndicatorTimeout flag, the client displays a sorry response to the user and stops the typing indicator. You can configure the sorry response using the defaultSorryMessage translation string.

    Check out the user guide for the default timeouts and the corresponding default messages.

  • Widget Name Configuration - We have introduced a new name configuration setting. This name configuration is used as a prefix to the CSS classes added to the widget. This uniquely identifies each widget if you have multiple instances of the chat widget on that same page, and helps us style them differently. The default value is 'oda-chat'.

What's Improved

  • Conversation Begin Position - The default conversation start position (configured with conversationBeginPosition) is now bottom.
  • Improved Pre-Built Themes - All the pre-built themes, DEFAULT, CLASSIC, and REDWOOD_DARK have been updated with new color shades with WCAG 2.1-compliant contrast.

What's Fixed

  • Long-Polling fallback for CSP WebSocket block - If the chat WebSocket connections are blocked due to CSP restrictions, the connection will fallback to long polling if the enableLongPolling flag set to true.
  • Prevent widget from going out of screen when it's positioned on left - If the widget is position on left side of the screen, the widget will expand to the right side of the button so that the widget is always visible.
  • Scrolling/Styling fix for multiple instances of the widget - You can now use multiple instances of the widget and scroll them independently. You can also customize the two instances separately using the name configuration.
